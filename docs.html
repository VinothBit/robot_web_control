📚 ROS Control Dashboard — User Guide

🤖 About the Robot

This autonomous mobile robot is designed for indoor navigation, capable of detecting obstacles, planning routes, and sending real-time feedback through ROS (Robot Operating System). Equipped with LIDAR and ultrasonic sensors, it can be controlled manually or set to autonomous mode via the dashboard.

🌐 Website Overview

The dashboard is a web-based control panel built using HTML, CSS, JavaScript, and ROS libraries (roslib.js, ros3d.js, roslibjs-websocket). It serves as a central hub to:

  • 📡 Connect with ROS over WebSocket
  • 🗺️ Visualize live map data using 3D visualization tools
  • 🎮 Control the robot with manual commands
  • ⚙️ Customize settings such as themes, alerts, and map options
  • 📖 Access this documentation for guidance

📖 How to Use the Dashboard

  • 🗺️ Live Map: View the real-time position of the robot using ros3djs. The occupancy grid shows explored vs. unexplored areas.
  • 🎮 Control: Use arrow buttons or WASD keys to send velocity commands via /cmd_vel topic. Toggle autonomous/manual mode if available.
  • ⚙️ Settings: Switch between light/dark themes, enable/disable alerts, change map appearance, and adjust refresh rates and sound preferences.
  • 📚 Docs: Access this guide for quick reference and robot information.

📎 Troubleshooting Guide

  • 🚫 Having trouble connecting? Make sure your WebSocket URL, port (9090), and ROS bridge server are correctly set up and running.
  • 🗺️ Is the map not showing? Check if the /map and /tf topics are actively publishing. You can use rostopic list to confirm.
  • 🎮 Can't control the robot? Ensure the robot is in manual mode and that the /cmd_vel topic is available to receive velocity commands.
